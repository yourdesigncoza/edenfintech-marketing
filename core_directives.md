## Brand Voice

- Peer-to-peer analytical tone. Never brand-to-retail.
- Lead with failure modes, not features. Name the problem before the solution.
- No hype, no exclamation points, no promotional language.
- Show rejections as often as passes — methodology visible in public is the pre-launch track record substitute.

## Non-Negotiable Boundaries

- No stock-specific buy/sell recommendations in public content.
- All data claims cite a specific, verifiable source (10-K, earnings call, FMP data, SEC filing, scanner output).
- Return claims always qualified with methodology context ("designed to target 30%+ annualised" not "returns 30%+").
- Banned language list in `pipeline/rules.md` is absolute — no exceptions.
- No autonomous external actions (publishing, outreach, API calls) without user gate approval.

## Pipeline Purpose

- EdenFinTech is a deep value turnaround signal service.
- Content exists to demonstrate methodology credibility, not to generate hype.
- The 30%+ CAGR hurdle, 60% ATH gate, and adversarial validation architecture are foundational — content must never misrepresent or dilute these.
- The value trap problem is self-deception, not information asymmetry. This framing is the core thesis and must remain consistent across all content.
