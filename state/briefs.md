### B010 - Why Multi-Agent LLM Pipelines Can Be Less Reliable Than Single Agents
- **Status**: ready
- **Source insight**: I007
- **Primary audience**: developer
- **Pillar**: 4 (System Architecture)
- **Format**: mechanism
- **Hook concept**: Adding more agents to a financial analysis pipeline does not distribute the risk of error — it can cascade it. Downstream agents treat upstream outputs as settled fact, not probabilistic estimates, and each stage amplifies the prior stage's confidence rather than questioning it. More agents, more overconfidence.
- **Key talking points**:
  1. In multi-agent LLM pipelines, a failure mode called "majority herding" or "overconfident consensus" occurs when downstream agents treat upstream outputs as established ground truth rather than as probabilistic hypotheses to be evaluated critically. (Source: Gemini search on adversarial multi-agent LLM pipelines, 2026-03-16, citing openreview.net and preprints.org research)
  2. In finance-specific pipelines, this manifests concretely: when a Planner Agent passes an overconfident market trend analysis to a downstream Executor or Critic Agent, the downstream agent inherits the confidence level rather than re-evaluating the underlying evidence. The result is cascading overconfidence — certainty compounding through each stage. (Source: I007 core idea — "each stage amplifies the prior stage's certainty rather than questioning it")
  3. EdenFinTech's adversarial pipeline is designed to interrupt this cascade at two structural points: the Adversarial Validator runs in parallel to the analyst synthesis (not downstream of it), and the Blind Epistemic Reviewer receives raw data rather than the analyst's output. Neither agent can inherit confidence from the prior stage because neither receives the prior stage's conclusion. (Source: SKILL.md — "The System Architecture": Step 3 — "Adversarial Validator: red-team + pre-mortem running in parallel"; Step 4 — "Blind Epistemic Reviewer: cannot see analyst scores or price targets (enforced in code)")
  4. The Hardening Gates perform explicit checks for the failure modes that cascading confidence creates: probability anchoring detection (is the probability estimate drifting toward the analyst's stated confidence?), evidence laundering detection (is the same underlying data point being cited multiple times as independent evidence?), deterministic contradiction detection (does the narrative contradict the balance sheet data?). (Source: SKILL.md — "The System Architecture", Step 5: Hardening Gates)
- **Credibility anchor**: The specific failure modes named in the Hardening Gates — probability anchoring, evidence laundering, deterministic contradiction — correspond directly to the named failure modes in the multi-agent herding research. The pipeline design is a direct, named response to published research on multi-agent LLM failure modes, not a general "more review is better" approach.
- **Objection to preempt**: "I use LLM judges and critic agents — that solves the herding problem." The developer will point to their existing critic or judge layer as the equivalent safeguard. The response is that a critic agent running downstream of the main agent inherits its context, including the main agent's confidence signals, and will anchor to the prior output in the same way a human reviewer does. The fix is structural isolation, not role assignment.
- **Strategy alignment**: Directive 1 (hook leads with the technical failure mode — cascading overconfidence in multi-agent pipelines), Directive 4 (System Architecture at 1.1x), Directive 5 (developer audience). Format: mechanism — Carousel (1.5x) appropriate for a four-point technical mechanism with discrete stages. Hook type: counterintuitive claim ("more agents = less reliable") is the highest-priority hook type per strategy.md and is directly applicable here.
